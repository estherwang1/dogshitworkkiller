# DogshitWorkKiller

用 AI 自动化杀掉重复性、格式化的"狗屎工作",把省下的时间留给真正有价值的事。

---

## 项目定位

**DogshitWorkKiller 是沉淀池,不是垃圾桶。**

- **一次性狗屎**:找 AI 当场处理完,不做成工具
- **重复出现的狗屎**(高频、稳定、可预测):沉淀到本项目里做成长期工具

判断标准:一件恶心的事被三次后,才考虑做成微应用。

---

## 项目的元比喻:造房子

理解本项目的心智模型:

> 我们想造一栋房子。
>
> 先要知道**什么是好房子**(**哲学**)——这是世界观和美学。有些是共识
> (房子要稳定),有些是个性(喜欢空中花园还是金碧辉煌)。
>
> 然后画**图纸**(**代码文件**,每行代码是砖块)把脑海里的房子落到现实。
> 图纸很多时,我们需要**目录**(**架构文档**)告诉我们每张图纸是干什么的,
> 不然找一张图纸要翻一百页。
>
> 画完图纸要施工,施工要遵循**标准**(**编码规范**),这样每个工人砌出来
> 的墙才不会风格各异。
>
> 施工完成后,房子里留下一堆**可调的旋钮**(**配置文件**)——温度、灯光、
> 门锁密码。住户可以调旋钮而不用动房子本身。
>
> 施工过程中,我们会不断做**设计决策**:这堵墙用砖还是用混凝土?地板用
> 木头还是瓷砖?这些决策当时有理由,时间一长容易忘,所以记进**改造档案**
> (**决策日志**),以后想改的时候能翻到当初的考虑。
>
> 每完成一个阶段,记一笔**施工日志**(**CHANGELOG**)——做了什么、改了
> 什么、拆了什么。三个月后回看就知道房子是怎么一步步成型的。
>
> 下一步要扩建哪个房间、装修哪面墙,写进**施工计划**(**plan**)——
> 和图纸目录分开,因为计划经常变,图纸不该跟着变。

## 快速上手

### 新建一个任务

复制 `templates/task_template/` 到 `tasks/` 下,改目录名,按模板里的
README.md 说明逐个填写文件。

### 跑已有任务

```bash
cd tasks/01_std_eval
# 编辑 config.yaml,填入输入输出路径和 LLM 地址
python runner.py
```

每个任务目录下有 README.md,说明该任务的用途、配置和运行方式。

### 依赖

项目需要 4 个 Python 包:

- openai
- python-docx
- pyyaml
- openpyxl

### 环境搭建

如果你的环境是内外网隔离 + 嵌入式 Python,参见 [ENVIRONMENT.md](docs/ENVIRONMENT.md),
里面有从零搭建的完整步骤。
